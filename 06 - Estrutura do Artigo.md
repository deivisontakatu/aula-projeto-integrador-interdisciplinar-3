# 📝 Guia de Escrita Científica para o Artigo do PI3

Este guia foca **exclusivamente na produção do artigo científico**, garantindo clareza, rigor acadêmico e integração entre as áreas do projeto.

---

# 🎯 Objetivo do Artigo

O artigo deve:

- Explicar **o problema industrial**
- Justificar a **solução proposta**
- Demonstrar **integração interdisciplinar**
- Apresentar **resultados técnicos e aplicados**
- Evidenciar **contribuição acadêmica**

> ⚠️ Não é apenas descritivo — precisa ser **analítico, técnico e crítico**

---

# 🧱 Estrutura Ideal do Artigo

## 1. Introdução

### O que deve conter:
- Contextualização (Indústria 4.0, IoT, digitalização)
- Problema real (ex: falta de integração industrial)
- Objetivo geral do trabalho
- Breve descrição da solução
- Organização do artigo

### Boas práticas:
- Comece do **macro → micro**
- Evite jargões sem explicação
- Seja direto (1 a 2 páginas)

---

## 2. Fundamentação Teórica

### Deve abordar:

- IoT Industrial
- Computação em Nuvem
- Sistemas Distribuídos
- UX Industrial
- Integração Vertical e Horizontal
- Metodologias Ágeis (Scrum)

### Boas práticas:

- Use **referências confiáveis (artigos, livros, normas)**
- Não copie — **interprete e conecte com o projeto**
- Sempre responda:
  > “Por que isso é relevante para minha solução?”

---

## 3. Metodologia

### O que explicar:

- Tipo de pesquisa (aplicada, experimental, etc.)
- Arquitetura do sistema
- Tecnologias utilizadas (MQTT, APIs, Cloud, Mobile)
- Organização via Scrum
- Divisão de responsabilidades (interdisciplinaridade)

### Boas práticas:

- Escreva no **passado**
- Seja claro e replicável
- Use diagramas (se possível)

---

## 4. Desenvolvimento da Solução

### Conteúdo esperado:

- Descrição técnica do sistema:
  - IoT
  - Backend
  - Cloud
  - Mobile
  - Interfaces industriais
- Fluxo de dados
- Integrações realizadas

### Boas práticas:

- Organize por **camadas do sistema**
- Evite código — foque em **arquitetura e funcionamento**
- Use figuras explicativas

---

## 5. Resultados

### O que apresentar:

- Funcionamento do sistema
- Testes realizados
- Evidências práticas
- Prints, métricas ou validações

### Boas práticas:

- Seja **objetivo**
- Mostre resultados reais (mesmo que simples)
- Não misture com discussão ainda

---

## 6. Discussão

### Aqui você deve:

- Analisar os resultados
- Relacionar com a teoria
- Mostrar ganhos da integração interdisciplinar
- Apontar limitações

### Boas práticas:

- Seja crítico (isso conta muito na nota)
- Evite repetir resultados — **interprete-os**
- Compare com outras abordagens

---

## 7. Conclusão

### Deve conter:

- Retomada do objetivo
- Principais resultados
- Contribuições do projeto
- Possíveis trabalhos futuros

### Boas práticas:

- Não introduza novos conceitos
- Seja direto e fechado

---

# ✍️ Boas Práticas de Escrita Científica

## 📌 Clareza e Objetividade

- Frases curtas
- Evite linguagem informal
- Evite “achismos”

❌ "A gente fez um sistema legal"  
✅ "Foi desenvolvido um sistema para monitoramento industrial em tempo real"

---

## 📌 Linguagem Técnica

- Use termos técnicos corretamente
- Defina conceitos na primeira vez que aparecerem

---

## 📌 Coesão e Coerência

- Um parágrafo = uma ideia
- Use conectivos:
  - “Além disso”
  - “Portanto”
  - “Dessa forma”

---

## 📌 Uso de Referências

- Sempre cite quando usar ideias externas
- Use padrão exigido (ABNT ou outro)

Exemplo:
> Segundo autor (ANO), IoT é…

---

## 📌 Padronização

- Mesma fonte e tamanho
- Títulos consistentes
- Figuras numeradas

---

## 📌 Evitar Erros Comuns

- ❌ Plágio
- ❌ Texto opinativo sem base
- ❌ Falta de ligação entre seções
- ❌ Excesso de prints sem explicação

---

# 🧠 Escrita Interdisciplinar (Ponto CHAVE do PI3)

Seu artigo deve mostrar que:

- As áreas **não são isoladas**
- Existe **integração real entre os módulos**

### Como fazer isso:

- Relacione partes:
  - IoT → Backend → Cloud → Mobile → Interface
- Explique dependências entre equipes
- Destaque decisões conjuntas

> Isso é o que diferencia um artigo comum de um **artigo forte**

---

# 📊 Dicas Avançadas (Para se destacar)

- Use diagramas claros de arquitetura
- Mostre fluxo de dados
- Compare com soluções tradicionais
- Destaque inovação (mesmo que simples)
- Escreva pensando:
  > “Alguém de fora entenderia meu sistema?”

---

# ✅ Checklist Final

Antes de entregar, verifique:

- [ ] O problema está claro?
- [ ] A solução está bem explicada?
- [ ] Existe base teórica suficiente?
- [ ] Os resultados são demonstrados?
- [ ] Há análise crítica?
- [ ] O texto está coeso?
- [ ] A interdisciplinaridade aparece?
